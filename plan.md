starSling CLI UI 计划（中文强化版，可直接交给 Codex 执行）

目标：把“像素风/蒸汽朋克/加粗/曲线”从主观描述改成硬约束 + 参考输出 + 可自动测试验收，确保 Codex 产出与期望一致。

⸻

0. 总目标与执行边界（必须遵守）
	•	本阶段只做 Go TUI UI 骨架 与 可渲染的静态 UI 示例，不做真实 CTP、真实下单、真实策略计算。
	•	UI 刷新频率：1 秒/帧 足够（不要追求高 FPS）。
	•	数据来源：先用 mock provider；未来 Python 端通过 IPC 推送数据。

⸻

1. 技术选型（固定，不给 Codex 自由发挥）
	•	选型：tcell + tview
	•	理由：布局、表格、panel、focus 管理成熟；多区块 dashboard 可控。
	•	颜色风格（蒸汽朋克/复古赛博）：
	•	颜色通过 tcell style 实现（背景深色 + 淡青/铜金点缀）。
	•	注意：颜色是加分项，不是主约束。主约束是字符画形状与布局。

⸻

2. 两个核心问题：根因与解决策略（写死）

2.1 Codex 写不对 STARSLING
	•	根因：让它“像素风”会触发 figlet/outline/自造字体，导致字形错误或拆词。
	•	解决：禁止生成字体，改为固定模板输出（snapshot）。
	•	Title 渲染不允许“算法生成字形”，只能输出固定 8 行模板。

2.2 Codex 画不对 parabola/曲线 + 星星
	•	根因：它倾向用块字符堆曲线或画成斜直线台阶；缺少“数学定义 + 区域限制”。
	•	解决：
	•	曲线必须按 明确曲线类型（二次 Bézier） + 锚点约束（起点/拱顶/终点） 绘制。
	•	曲线区域 禁止块字符（█▓▒░），只允许细线字符。
	•	Logo 画布固定高度 + 宽度上限，避免“巨大化”。

⸻

3. Main Screen（必须实现且可验收）

3.1 Logo 组件：材质严格分离

Logo = 上方曲线 + 星星 + 下方标题（STARSLING 像素字）

3.1.1 Logo 画布（防止画太大）
	•	RenderLogo(width int, frame int) []string 必须返回 固定 18 行。
	•	logoWidth = min(width, 110)：最大宽度 110 列。
	•	这 18 行内部再分：
	•	上 10 行：curve area（曲线+星星）
	•	下 8 行：title area（标题）

3.1.2 Title（只允许 chunk，且必须写对）
	•	Title 文本固定：STARSLING（全大写，9 个字母）。
	•	UI 中可额外用普通文本显示 StarSling（不属于 title area 约束），但像素块标题必须是 STARSLING。
	•	Title Area 只能使用字符：█ 和空格。（可选 ▓，但默认只用 █）
	•	禁止：- _ | / \\ . : + * 等描边字符。
	•	禁止 figlet、禁止自造字体。
	•	Title 尺寸：固定 8 行。
	•	Title 必须与下方参考模板逐行一致（允许行首尾空格差异，但中间结构必须一致）：

3.1.3 Curve（只允许细线字符，必须是 parabola/弧线）
	•	Curve Area 严禁使用 █▓▒░（这些只给标题用）。
	•	Curve 允许使用细线字符：─ ╱ ╲ │ •，以及 Braille 点阵字符（U+2800–U+28FF）用于高分辨率曲线。
	•	曲线必须是 parabolic/ballistic curve，不能是直线。

曲线定义：二次 Bézier（必须按此公式）
令 W = logoWidth，H = 10（curve 区高度），锚点：
	•	P0 起点：(x=round(0.12*W), y=round(0.80*H))
	•	Pm 控制点（拱顶更高）：(x=round(0.60*W), y=round(0.08*H))
	•	P1 终点：(x=round(0.88*W), y=round(0.05*H))

注：使用 Braille 高分辨率时，实际计算应在子像素网格完成：
	•	subW = 2*W，subH = 4*H
	•	在 subW/subH 上应用同样比例后再 round，这样才能体现细微抬升。

对 t 从 0 到 1 采样 N 次（N = 2*W）：
	•	x(t) = (1-t)^2*P0.x + 2*(1-t)*t*Pm.x + t^2*P1.x
	•	y(t) = (1-t)^2*P0.y + 2*(1-t)*t*Pm.y + t^2*P1.y

在画布上 plot (round(x), round(y))，字符使用 ·。

约束：曲线线宽固定 1 字符；曲线的“亮度/渐变”只能用 tcell.Style 颜色实现，禁止用 █▓▒░ 堆叠实现。

重要验收点（防止画成斜线台阶）
	•	apex 必须出现在 x ≈ 60%W 附近，并且 y 最小（最高点）。
	•	起点 y 要明显大于终点 y（左低右高趋势）。
	•	终点必须低于 apex（否则像一直上坡）。

3.1.4 Star（星星：先静态正确，再做轻微动画）
	•	星星位置：固定在 P1（终点）附近。
	•	静态帧必须明显：使用 ✶（首选），fallback *。
	•	动画（后续）：1 秒 tick 交替显示两种星形：
	•	帧 A：✶
	•	帧 B：✸ 或 *
	•	曲线不允许用块字符实现“发光”；发光靠颜色（tcell style）或少量 . 点缀。

4. Live market data 界面（布局不变，关键点写清）
	•	左 60%：
	•	上 70%：行情表格（可选行高亮）
	•	下 30%：日志滚动
	•	右 40% 三等分：
	1.	forward curve + vix（占位图线）
	2.	option chain（占位）
	3.	unusual option trades（信息窗口）：最新在最上方，向下滚动；列表而非图线

刷新 1 秒 tick；unusual trades 用 ring buffer，push-front。

5. 必须实现的验收测试（防 Codex 自由发挥）

5.1 Title 验收（解决字母错误）
	•	Title area（8 行）必须：
	1.	只包含 █ 和空格
	2.	与参考模板逐行一致（snapshot test）

5.2 Curve 验收（解决曲线错误）
	•	Curve area（上 10 行）必须：
	1.	不包含 █（避免 chunk 画 curve）
	2.	至少包含 20 个 ·（或 .）
	3.	apex 检测：在 x ∈ [0.50W, 0.70W] 范围内的最小 y 必须小于：
	•	起点 y - 2
	•	终点 y - 1
