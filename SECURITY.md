# Security Policy

## Supported Scope

`starSling` is currently in an MVP / prerelease stage. Security fixes are handled on a best-effort basis.

## Reporting a Vulnerability

- Do not open a public GitHub issue for suspected vulnerabilities.
- Report the issue privately to the maintainer at `muzexlxl@foxmail.com`.
- Include reproduction steps, affected version/commit, impact, and any suggested mitigation.

## Response Expectations

- Initial triage target: within 7 days.
- Remediation timelines depend on severity, reproducibility, and release impact.

## Handling Notes

- Please avoid sending live credentials or production secrets unless strictly necessary.
- If sensitive data was exposed, rotate it before or immediately after reporting.
